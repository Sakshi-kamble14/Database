# 📚 Database Theory

## 📖 Overview

Database theory focuses on the principles and concepts used to design, manage, and optimize databases. It provides the foundation for storing, retrieving, and manipulating data efficiently and securely.

---

## 🧠 Core Concepts

### 1. Data and Information

* **Data**: Raw facts (e.g., numbers, text)
* **Information**: Processed data with meaning

---

### 2. Database

A database is an organized collection of data that can be easily accessed, managed, and updated.

---

### 3. Database Management System (DBMS)

A DBMS is software that interacts with users and applications to manage the database.

* Examples: MySQL, PostgreSQL, Oracle

---

### 4. Data Models

Define how data is structured and stored:

* Hierarchical Model
* Network Model
* Relational Model
* Object-Oriented Model

---

### 5. Relational Model

* Data is stored in **tables (relations)**
* Tables consist of **rows (tuples)** and **columns (attributes)**
* Each table has a **primary key**

---

### 6. Keys in Database

* **Primary Key**: Uniquely identifies a record
* **Foreign Key**: Links two tables
* **Candidate Key**: Possible primary keys
* **Composite Key**: Combination of multiple columns

---

### 7. Normalization

Process of organizing data to reduce redundancy and improve integrity.

* **1NF (First Normal Form)**: No repeating groups
* **2NF**: No partial dependency
* **3NF**: No transitive dependency

---

### 8. Relationships

Defines how tables are connected:

* One-to-One
* One-to-Many
* Many-to-Many

---

### 9. Constraints

Rules applied to data:

* NOT NULL
* UNIQUE
* PRIMARY KEY
* FOREIGN KEY
* CHECK

---

### 10. Transactions

A transaction is a sequence of operations performed as a single unit.

#### ACID Properties:

* **Atomicity**: All or nothing
* **Consistency**: Maintains rules
* **Isolation**: Independent execution
* **Durability**: Permanent changes

---

### 11. Indexing

Improves data retrieval speed by creating a data structure for quick lookup.

---

### 12. SQL (Structured Query Language)

Used to interact with relational databases:

* DDL (Data Definition Language)
* DML (Data Manipulation Language)
* DCL (Data Control Language)
* TCL (Transaction Control Language)

---

## 🎯 Importance of Database Theory

* Ensures efficient data storage
* Reduces redundancy
* Improves data integrity
* Enables scalability
* Enhances performance

---

## 📌 Conclusion

Database theory forms the backbone of modern applications. Understanding these concepts is essential for designing robust, scalable, and efficient database systems.
